# 浪潮与根——技术世界里的学习、创新与立足

技术无穷无尽，领域茫茫多，而且越变越快。前面谈《工程技术》，讲的是怎么把一项技术做成立得住的产品，是造物与落地那一面。这篇要谈另一面，也是更让人焦虑的一面：在这样一个技术多到学不完、又快到追不上的世界里，一个人，推而广之一家企业、一个国家，该怎么学，怎么站上前沿，怎么创新，又靠什么立足。

先从一个让人不安的现象说起。

## 一、一个让人不安的现象

我们常被告诫，要选定一个领域，然后坚守它，因为选择大于努力。这话不错。可现实里另有一番景象：真正让一个人安身立命、让一家企业拿到市场的，往往不是那些需要熬很多年的硬核技术，而是踩中了某一波浪潮的应用技术。搜索的时代，广告的时代，推荐系统的时代，深度学习的时代，如今是大模型的时代，一浪接着一浪，每一浪都把一批人和公司抬了起来。而这些高价值的机会又显得可遇不可求，像是撞上的，而不是规划出来的。于是悖论就来了：既然要选一个领域死守，可值钱的浪潮又一波波地换、还撞大运似的难求，那么"坚守"到底守的是什么？

## 二、为什么高价值的技术可遇不可求

先弄清这些机会为什么这么难求。根子在于，数字时代的技术不依赖物质。一套软件、一个算法，不受产能、物料、地理的拖累，可以近乎零成本地复制，一夜之间铺向全球，还叠着越多人用就越好用的网络效应。这带来两个后果。一是爆发极快、赢家通吃，一个新范式打开的窗口，很短时间内就被瓜分干净。二是技术周期被大大压短，从前一项主导技术能管上几十年，如今隔几年就翻出一个新范式。两者叠在一起，能不能赶在一个新范式刚打开、红利还没分完的时候踩进去，就在很大程度上成了时机问题，它难以靠多年的硬核积累稳稳地规划出来，这才显得可遇不可求。这和材料、制造、器件那些依赖物质、周期漫长、壁垒厚重的硬核技术，恰好是两个样子。

## 三、坚守的是根，选择的是浪

明白了这一层，悖论也就解开了。关键在于分清，坚守的到底是什么。

坚守的不该是某一项具体技术。具体技术本就一波波起落，死守一项，迟早会被新浪潮拍在沙滩上。真正该守的，是那个不变的根：你所在的那个域，它底下的现象，这一行里多年不变的核心组件，以及一身能拆解、能迁移、能快速上手新东西的本事。而选择，选的是浪，是赛道，是时机，是那个可遇不可求的应用窗口。说选择大于努力，是因为选择决定了你站在哪条浪上、你的力气有没有杠杆：踩对一浪，回报是成倍的；方向错了，再使劲也是事倍功半。

但这两件事并不打架，恰恰相反，能一次次踩对浪潮，靠的正是守住了那个根。一个握着底层现象、熟悉核心组件、又善于迁移的人，才能在搜索、推荐、深度学习、大模型的每一浪里都迅速上手。所以坚守，是练就那身能反复骑浪的本事；选择，是每一次决定骑哪一道浪。真正的高手，是拿一个不变的根，去接住一个个变化的浪。

## 四、怎么学：把功夫下在那个不变的根上

那么这个根，该怎么练？先说学习。

最常见的错，是把学习当成求全，想着先把一个领域的知识点都学完，再去做事。可知识点是无穷的，越想学完越焦虑，而且死记下来的东西，换个场景多半就失效了。正确的做法是反过来，把学习当成解决问题、把控趋势的工具：带着一个具体的问题，先学解决它所需要的那最少的一点东西，用完再复盘，让知识随着一个个问题一轮轮地长出来。学的过程里要紧的，不是记住技巧本身，而是从技巧里提炼出那条可以迁移的核心逻辑；面对纷乱的变化，也不必去预测一切，只要抓住一两个真正影响趋势的关键变量就够了。

往大处看，这就是为自己种一棵活的知识树：树根是底层的现象与通用的原理，树干是一个域里多年不变的核心组件，枝叶才是那些具体的工具与框架。学习的方向永远是从根到叶，而不是反过来在枝叶里打转。把八成的力气下在那个恒久不变的核心组件库上，而不去追逐那些很快就会过时的新名词，因为追新名词像是不停地换一张很快作废的地图，而吃透了核心组件，才算攥住了一个不变的指南针。这棵树，就是前面说的那个根。

## 五、怎么创新：从组合到原始

学会了、站稳了，接下来是创新。创新并不是什么不可捉摸的灵感，它其实分两种，难度与价值都不一样。

一种是组合创新。技术从来都是已有技术的组合，所以最常见的创新，就是把现成的构件重新搭配，或者把一个老问题搬到一个新的域里去解。机械钟表换成石英，要解决的还是计时这件老事，原理没变，可整个形态、成本与精度都被重写了。这种创新是可以拆解、可以训练的，它是技术日常的引擎。

另一种更深，是原始创新。它不在已有的构件里打转，而是往源头去：捕捉一个还没有被人驯服的新现象，或者在两个看似毫不相干的领域之间，头一回认出同一根结构。前者给技术添了新的基因，后者把一个领域的结构迁到另一个领域，开出前所未见的可能。技术演化最深的那股活水，正在这里。

而原始创新靠的是什么？是三样彼此相连的本事：捕捉现象，看见结构，以及用相似相异认出同构。说到底，这就是洞察，是透过纷乱的表象，看见底下那根可以复用的骨架。捕捉现象、看见结构、相似相异，既是这本书反复讲的世界底层原理，也正是人做出原始创新的那把钥匙。

## 六、一个活例：大模型与 agent

这几句话听着抽象，拿眼下最新的一浪，也就是大模型与 agent，来看就具体了。

先说"捕捉现象其实就是认出结构"这件事，大模型是最好的例子。它捕捉的所谓"现象"，是海量数据尤其是语言里蕴含的统计结构，是人类知识以文字沉淀下来的那套可压缩的模式；所谓预测下一个词，本质就是在捕捉并复现这套结构。而它的工作机制，骨子里就是相似相异：注意力机制是按相似度给信息加权、检索、再组合，是相似相异的大规模工程化；词嵌入把万物映射进一个高维空间，意思相近的彼此相聚，是"相似"的几何化。这里要补一句分寸，它逼近的，是已经被人类记录下来的那层结构，离世界本身那层更深的原理，还隔着真实实验与因果这两关。但即便如此，它的成功也已经给"本质是结构、相似相异是认知的核心"这条原理，投下了极重的一票。它内部这几年的演化也在重演技术的老规律：混合专家把一个大模型拆成许多按需激活的专家，是把模块化组合搬进了模型内部；推理模型把"想"的算力挪到了回答的当口；上下文窗口也已经撑到了百万级。

再说 agent，它是组合创新与新域形成的眼前活例。把大模型这个新构件，组合进工具调用、记忆、规划与多步执行里，就组合出了"自主把一件事做完"的新功能，这等于给大模型接上了手脚，把它从"会说"推向了"会做"。而围绕它，一个新的域正在成形：模型上下文协议为 agent 提供了标准化的工具接口与组合规则，正是阿瑟所说的那种"域"的当代建造，到二〇二六年，它的月下载量已近亿，公开的服务器过万，几家大厂都已支持；agent 与 agent 之间如何委派、协作，又长出了配套的协议，多 agent 的系统随之兴起。从大模型捕捉结构，到 agent 把它组合成会做事的系统，再到协议长成一个新的域，这正是"现象捕获、递归组合、域的形成、社会牵引"这条演化规律，在数字经济里被极速地演了一遍。

而这一浪还逼出了一个值得记住的反转。大模型最擅长的，恰恰是复现和重组已有的结构，于是它把组合创新的成本压到了极低。可也正因如此，它反过来把原始创新的价值顶得空前之高，因为捕捉一个还没有被任何文字记录过的新现象、在无人之境看见一条新结构，恰恰是大模型这台困在已有语料里的机器做不到、而能下场做真实实验、能亲历现象的人仍然守得住的那块领地。大模型越强，这把原始创新的钥匙，就被衬托得越稀缺、越珍贵。

## 七、技术是什么，又该放在哪里谈

最后退一步，看看"技术"这个东西本身。

谈到这里，留心的人会发现，这套关于技术本质的说法，骨子里其实是几种哲学在技术上的落地，尽管它们从没被点名。说技术是对现象的捕获与编程、说现象才是技术真正的源头，这是现象学"回到现象本身"的技术版，是把自然现象带入人的目的、加以揭示。而说技术是生产力的核心载体、说经济不过是技术的外在表达、说一个技术方案催生新的经济结构又催生新的问题再呼唤更新的技术，这分明是马克思那套"生产力决定生产关系"的具体化。技术本质论，是现象学与马克思的一次没有点名的合流。

也有人会问，谈技术为什么不去谈生产力、生产关系、商品这些更熟悉的概念。一来是层次不同，那些是政治经济学的宏观范畴，讲的是社会结构，而这里说的技术，是技术系统自身演化逻辑这个中观的层面，两者不在一层，不必混着说。二来是，技术哪怕脱开那套框架单独来看，依然是一个枢纽，它向上承接着经济、国家竞争力与社会发展，向下连着个人的安身立命、学习与创新，正是连起宏观社会与微观个人的那个节点。所以单就"技术"来谈，既谈得通，也值得谈。

## 收束

技术无穷无尽，领域茫茫多。学一项具体技术，多数时候把它当一个黑箱用就够了，不必深究底下那些抽象原理。可一旦你要高屋建瓴地去学习、去创新、去规划方向，就得退到技术的本质与演化那一层，抓住那三个不变的问题——它在哪个域，由哪些构件组合，又驯服了哪个现象——拿这套眼力当地图。这正是经验与理论、现成的桥与底下河床的分别：具体技术像经验、像一座现成的桥，平时尽够用；技术的本质像理论、像河床，只在你要规划、要创新的时候才需要下去。

而那个根，那条以不变应万变的线，说到底就是结构与相似相异这套底层的眼力。看穿了注意力不过是相似相异、agent 不过是组合创新、新协议不过是在建一个新的域，你就不会被层出不穷的新名词唬住，能比别人更快上手，也能更冷静地判断哪一波才是真红利。技术的浪潮一浪接着一浪，没有谁追得完；真正能追的，从来不是每一道浪，而是练就那身一看就认出浪的形状的本事。
